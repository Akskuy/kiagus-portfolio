# SECTION PLAN — INTERACTIVE SCENE JOURNEY

## 1. Main Principle

This portfolio is organized into 8 scenes.

The term "slide" may still be used for file naming and references, but the final user experience must not feel like normal vertical slides.

The website must not feel like:

* stacked sections
* simple scroll-down page
* normal landing page
* one section fading into the next
* generic resume website
* plain text and cards

The website must feel like:

* connected game scenes
* interactive digital rooms
* mission progression
* data-science world exploration
* premium pixel UI system
* object-based portfolio journey

Each scene must have:

1. Scene identity
2. Main hero object
3. Live background/environment
4. Title-only interactive panels
5. Creative panel reveal behavior
6. Idle motion
7. Hover/tap motion
8. Transition gate to the next scene

Scroll is not just page movement.

Scroll should act as scene control.

The user should feel that they are progressing through a designed digital system.

---

## 2. Global Scene Flow

The portfolio journey:

1. Scene 1 — Identity Room
2. Scene 2 — Capability Command Center
3. Scene 3 — Knowledge Library
4. Scene 4 — Experience District
5. Scene 5 — Project Archive
6. Scene 6 — Achievement Analytics
7. Scene 7 — Knowledge Map
8. Scene 8 — Future Laboratory

Every scene should visually connect to the next scene through a transition gate.

Do not create disconnected sections.

Do not use ordinary page jumps.

Do not use plain fade between scenes.

---

## 3. Global Scene Phase Structure

Every scene should follow this internal structure:

## Phase A — Scene Entry

The scene begins from the previous scene's transition object.

Examples:

* identity data becomes command center
* terminal fragments become knowledge shelves
* book page becomes city map
* building compresses into cartridge
* project scanner becomes achievement analytics
* trophy reflections become nodes
* data core opens future lab

## Phase B — Scene Boot

The scene activates.

Examples:

* lights turn on
* monitor wall flickers
* UI panels boot
* terminals start blinking
* data lines begin moving
* scanlines sweep
* status LEDs activate
* background depth layers become visible

## Phase C — Main Interactive State

The scene becomes usable.

Required:

* hover/tap states
* title-only panels
* creative reveal animation
* selected/active state
* detail panel content
* readable portfolio information
* ambient motion

## Phase D — Transition Gate

A specific visual object prepares the next scene.

Examples:

* lanyard card sends identity data
* terminal breaks into data blocks
* book page extrudes into city
* building compresses into cartridge
* scanner output becomes achievement dashboard
* trophy shatters into nodes
* nodes collapse into data core

## 14. UI Density Rule Per Scene

Every scene should have rich UI details.

However, do not make the scene unreadable.

Use this priority:

1. Clear information
2. Strong scene identity
3. Rich game-like UI
4. Controlled animation
5. Responsive layout

Do not leave empty dark spaces without purpose.

If content is short, enrich the scene with:

* environment objects
* micro labels
* animated data lines
* status LEDs
* decorative but meaningful panels
* icon tiles
* monitor rows
* data chips
* glowing dividers
* subtle scanline frames
* small system captions

Do not use plain filler.

Everything should feel part of the interface system.

## 16. Acceptance Criteria Per Scene

Before a scene is considered complete, check:

1. Does it follow the clean target composition?
2. Does it avoid rendering reference images as static background?
3. Does it exclude concept-board annotation text?
4. Does it feel like a scene, not a section?
5. Are important elements alive?
6. Are small UI elements visually themed?
7. Are panels not plain rectangles?
8. Are title-only panels revealed creatively?
9. Are hover/tap states meaningful?
10. Is the transition gate prepared?
11. Is motion more advanced than fade/slide?
12. Is text readable?
13. Does it fit at 100% browser zoom?
14. Does it avoid fake claims?
15. Does it feel premium, not childish?
16. Does it feel difficult to build?

Final goal:

The portfolio should feel like a premium interactive data-science game journey.
